# Originality Statement

The AI Incident Registry (AICI) represents a category-creating initiative to catalog AI authored code failures.

This document formally states that the standard, taxonomy, classification schemas, and associated tooling within this repository are original works. 

1. **No Copied Text**: No text, definitions, or documentation has been copied from MITRE CVE, OSV, AVID, AI Incident Database, or CWE documentation.
2. **Original Schema**: The YAML schema structure and taxonomy defined in `SPEC.md` and `TAXONOMY.md` were designed from first principles to address the unique context of AI-authored code failures, and do not reproduce or derive from the CVE JSON 5.0 schema or the AVID taxonomy.
3. **Identifier Format Structure**: The `AICI-YYYY-NNNN` identifier format adopts an industry-standard structural convention (similar to CVE or advisory formats) for sequential tracking. This uncopyrightable structural convention is heavily utilized across the cybersecurity industry and does not infringe upon any existing intellectual property.
4. **Differentiation**: The scope of this registry is uniquely defined (failures of AI-authored code in production or deployment), distinguishing it explicitly from models observing AI harms or product vulnerabilities.

For further details on how this project complements existing databases, see `PRIOR_ART.md`.
